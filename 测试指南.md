# 垃圾 BP 过滤器 - 测试指南

## 快速测试步骤

### 1. 检查环境配置

```bash
# 检查 .env 文件是否存在
cat .env

# 确保以下配置已填写：
# MINIMAX_API_KEY=sk-xxx（必填）
# SERPER_API_KEY=xxx（可选，不填则使用 Mock 模式）
```

### 2. 安装依赖

```bash
# 安装 Node.js 依赖
npm install

# 安装 Python 依赖（PDF 提取）
pip3 install pymupdf

# 可选：安装 OCR 支持（扫描版 PDF）
pip3 install pdf2image pytesseract Pillow
# macOS: brew install poppler tesseract tesseract-lang
# Ubuntu: apt-get install poppler-utils tesseract-ocr tesseract-ocr-chi-sim
```

### 3. 启动服务

**方法 1：一键启动（推荐）**
```bash
# Mac/Linux
chmod +x start.sh
./start.sh

# Windows
# 双击 start.bat
```

**方法 2：手动启动**
```bash
# 终端 1：启动后端
cd server
npm install
node index.js

# 终端 2：启动前端（开发模式）
cd client
npm install
npm start
```

### 4. 访问应用

打开浏览器访问：`http://localhost:3000`

## 常见问题排查

### 问题 1：解析异常（已修复）

**症状：** 分析过程中显示"解析异常"，多次重试失败

**原因：** LLM 输出格式不标准，JSON 解析失败

**解决方案：** 
- ✅ 已在 v2.1 版本修复
- 增强了 JSON 提取和容错能力
- 添加了 3 次重试机制

**验证修复：**
```bash
# 运行 JSON 解析测试
node test_json_parse.js

# 预期：所有测试用例通过
```

### 问题 2：PDF 提取失败

**症状：** "PDF 解析失败" 或 "提取的文本过短"

**可能原因：**
1. PDF 已加密
2. PDF 是扫描版但未安装 OCR
3. PDF 文件损坏
4. PDF 内容确实很少

**解决方案：**

```bash
# 1. 检查 PDF 是否可以正常打开
open test.pdf  # macOS
xdg-open test.pdf  # Linux

# 2. 手动测试 PDF 提取
python3 scripts/extract_pdf.py test.pdf

# 3. 如果是扫描版，安装 OCR
pip3 install pdf2image pytesseract
brew install poppler tesseract tesseract-lang  # macOS

# 4. 检查提取的文本长度
python3 scripts/extract_pdf.py test.pdf | wc -c
# 应该 > 500 字符
```

### 问题 3：API Key 无效

**症状：** "API 调用失败" 或 401/403 错误

**解决方案：**

```bash
# 1. 检查 .env 文件
cat .env | grep MINIMAX_API_KEY

# 2. 验证 API Key 格式
# 应该是 sk-xxx 格式

# 3. 测试 API Key（使用 curl）
curl https://api.minimax.io/anthropic/v1/messages \
  -H "x-api-key: $MINIMAX_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "MiniMax-M2.5",
    "max_tokens": 100,
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

### 问题 4：搜索功能不可用

**症状：** 显示 "Mock 模式" 或 "搜索未启用"

**说明：** 这不是错误，是正常的降级行为

**影响：** AI 仍可分析，但无法联网验证 BP 诉求

**解决方案（可选）：**

```bash
# 1. 注册 Serper.dev 账号
# https://serper.dev
# 免费额度：2500 次搜索

# 2. 获取 API Key

# 3. 添加到 .env
echo "SERPER_API_KEY=你的key" >> .env

# 4. 重启服务
```

### 问题 5：端口被占用

**症状：** "Error: listen EADDRINUSE: address already in use :::3001"

**解决方案：**

```bash
# 查找占用端口的进程
lsof -i :3001  # macOS/Linux
netstat -ano | findstr :3001  # Windows

# 杀死进程
kill -9 <PID>

# 或者修改端口
echo "PORT=3002" >> .env
```
