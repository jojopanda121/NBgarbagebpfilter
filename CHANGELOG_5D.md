# 五维标准化评分系统 - 更新说明

## 更新内容

本次更新将原有的10个评分维度重构为5个标准化维度，采用投资机构级别的评分公式，确保评分的客观性和一致性。

## 主要变更

### 1. 评分维度合并（10 → 5）

**原10个维度：**
- 市场规模真实性
- 估值合理性  
- 技术可行性
- 技术先进性
- 竞争壁垒
- 创始团队匹配度
- 团队完整性
- 痛点刚性
- 入场时机
- 资本适配度

**新5个维度：**

1. **时机与天花板**（权重 20%）
   - 合并：市场规模 + 入场时机
   - 公式：`S1 = min[100, (20 × log10(TAM+1) + 2 × CAGR)]`

2. **产品与壁垒**（权重 25%）
   - 合并：技术可行性 + 竞争壁垒
   - 公式：`S2 = 0.4 × (TRL/9 × 100) + 0.6 × SC`

3. **商业验证与效率**（权重 35%）
   - 合并：商业模式 + 用户增长 + 财务健康度
   - 公式：`S3 = 0.7 × f(LTV/CAC) + 0.3 × min(100, Margin×100)`

4. **团队基因**（权重 20%）
   - 保留：团队匹配度 + 股权结构
   - 公式：`S4 = min(100, Exp×10) - Penalty(Equity)`

5. **外部风险与交易条件**（作为乘数）
   - 合并：政策风险 + 估值合理性
   - 公式：`V5 = Policy_Risk × Discount(Valuation_Gap)`

**总分公式：**
```
Total_Score = (S1×20% + S2×25% + S3×35% + S4×20%) × V5
```

### 2. 评级标准更新

| 评级 | 分数 | 标签 | 说明 |
|-----|------|------|------|
| **A** | ≥85 | 推荐投资 (Fast Track) | 立刻推进，24小时内约见创始人 |
| **B** | 75-84 | 谨慎推荐 (Proceed with DD) | 安排面聊，核心考察团队 |
| **C** | 60-74 | 可以跟进 (Keep In View) | 商业逻辑待验证，关注签约 |
| **D** | <60 | 建议放弃 (Reject / Archive) | 结构性死亡，伪需求或烧钱无底洞 |

## 核心优势

1. **标准化公式**：每个维度都有明确的数学公式，减少主观判断
2. **防作弊机制**：使用对数函数压制TAM夸大，分段函数评估LTV/CAC
3. **一票否决**：第五维度作为乘数，可直接否决高风险或离谱估值的项目
4. **行业对标**：强制与行业平均水平对比，防止盲目乐观
5. **简洁清晰**：从10维度简化为5维度，更易理解和沟通

## 使用方法

### 启动服务

```bash
# 安装依赖（如未安装）
npm install
cd server && npm install && cd ..

# 构建前端
npm run build

# 启动服务器
cd server && npm start
```

服务将在 `http://localhost:3001` 启动。

### 环境变量

确保 `.env` 文件配置正确：

```bash
MINIMAX_API_KEY=your_minimax_api_key
SERPER_API_KEY=your_serper_api_key  # 可选，用于联网搜索
PORT=3001
```

## 注意事项

1. **数据提取**：AI会尽力从BP中提取评分所需数据，如果BP中未明确提及，会根据行业常识推断
2. **联网验证**：搜索结果用于校准BP中可能夸大的数据
3. **估值对比**：系统会自动计算BP估值与行业平均的溢价倍数
4. **一票否决**：如果估值溢价超过3倍或涉及政策禁止领域，总分会大幅降低
