# 五维标准化评分系统说明

## 概述

本系统将原有的10个评分维度合并为5个核心维度，采用标准化的评分公式，确保评分的客观性和可比性。

## 五个维度

### 第一维度：时机与天花板（权重 20%）
**合并维度：** 市场规模 + 入场时机

**输入变量：**
- `TAM`: 目标可触达市场规模（单位：十亿人民币或亿美元）
- `CAGR`: 行业预期年复合增长率（%）

**计算公式：**
```
S1 = min[100, (w_a × log10(TAM + 1) + w_b × CAGR)]
```
- `w_a = 20`: TAM权重系数
- `w_b = 2`: CAGR权重系数

**说明：** 使用对数函数压制TAM数据的夸大，防止创始人无脑注水。

---

### 第二维度：产品与壁垒（权重 25%）
**合并维度：** 技术可行性 + 竞争壁垒

**输入变量：**
- `TRL`: 技术就绪水平（Technology Readiness Level, 1-9级，9为可量产）
- `SC`: 客户转换成本评分（Switching Cost, 0-100分）

**计算公式：**
```
S2 = 0.4 × (TRL / 9 × 100) + 0.6 × SC
```

**TRL等级说明：**
- 1-3级：概念/实验室阶段（PPT阶段）
- 4-6级：原型/小试阶段
- 7-9级：中试/量产阶段

---

### 第三维度：商业验证与效率（权重 35%）
**合并维度：** 商业模式 + 用户增长/留存 + 财务健康度

**输入变量：**
- `Ratio`: LTV/CAC比值（客户生命周期价值/客户获客成本）
- `Margin`: 毛利率（0-1之间的小数）

**计算公式：**
```
f(Ratio) = {
  0                      if Ratio ≤ 1
  50 × (Ratio - 1)       if 1 < Ratio < 3
  100                    if Ratio ≥ 3
}

S3 = 0.7 × f(Ratio) + 0.3 × min(100, Margin × 100)
```

**说明：** 
- LTV/CAC < 1：做慈善（0分）
- LTV/CAC > 3：好生意（100分）
- 1 < LTV/CAC < 3：线性增长

---

### 第四维度：团队基因（权重 20%）
**保留维度：** 团队匹配度

**输入变量：**
- `Exp`: 核心创始人行业相关经验（年）
- `Equity`: 最大股东持股比例（%）

**计算公式：**
```
Penalty(Equity) = {
  50     if Equity < 30%
  20     if 30% ≤ Equity < 50%
  0      if Equity ≥ 50%
}

S4 = min(100, Exp × 10) - Penalty(Equity)
```

**说明：** 早期项目如果大股东持股不足50%，会面临后续融资困难。

---

### 第五维度：外部风险与交易条件（作为乘数）
**合并维度：** 政策/监管风险 + 估值合理性

**输入变量：**
- `Policy_Risk`: AI判定的政策违规风险等级（0为极高风险，1为无风险）
- `Valuation_Gap`: BP叫价与行业可比公司中位数的溢价倍数

**计算公式：**
```
Discount(x) = {
  1.0     if x ≤ 1.5    (溢价在合理范围内)
  0.5     if 1.5 < x < 3 (估值过高，需重度砍价)
  0       if x ≥ 3      (离谱叫价，直接否决)
}

V5 = Policy_Risk × Discount(Valuation_Gap)
```

**说明：** 此维度作为乘数，可以一票否决项目。

---

## 总分计算

```
Total_Score = (S1 × 20% + S2 × 25% + S3 × 35% + S4 × 20%) × V5
```

## 评级标准

| 分数范围 | 评级 | 标签 | 建议行动 |
|---------|------|------|---------|
| ≥ 85分 | **A** | 推荐投资 (Fast Track) | 立刻推进：24小时内约见创始人，启动业务尽调和财务尽调，开始建模 |
| 75-84分 | **B** | 谨慎推荐 (Proceed with DD) | 安排面聊，核心考察团队，在财务数据中申请行权测试，并开始估价 |
| 60-74分 | **C** | 可以跟进 (Keep In View) | 商业逻辑穿透没准，但缺乏数据验证。建议梳理POC，关注签约，再评估 |
| < 60分 | **D** | 建议放弃 (Reject / Archive) | 结构性死亡：伪需求，烧钱无底洞，股权结构混乱，严重高估 |

## 数据提取与验证流程

### 第1步：数据提取（Agent A）
从BP中提取评分所需的原始数据：
- TAM, CAGR, TRL, SC
- LTV, CAC, Ratio, Margin
- Exp, Equity
- Policy_Risk, BP_Valuation, BP_Revenue

### 第2步：联网取证
- 搜索验证BP中的声明
- 查询行业平均估值数据（AkShare）
- 获取可比公司数据

### 第3步：数据验证与评分（Agent B）
- 验证BP数据真实性
- 校准被夸大的数据
- 补充缺失数据
- 计算估值溢价倍数
- 应用评分公式

## 与旧系统的对比

### 旧系统（10维度）
1. 市场规模真实性
2. 估值合理性
3. 技术可行性
4. 技术先进性
5. 竞争壁垒
6. 创始团队匹配度
7. 团队完整性
8. 痛点刚性
9. 入场时机
10. 资本适配度

### 新系统（5维度）
1. **时机与天花板**（市场规模 + 入场时机）
2. **产品与壁垒**（技术可行性 + 竞争壁垒）
3. **商业验证与效率**（商业模式 + 增长 + 财务）
4. **团队基因**（团队匹配度 + 股权结构）
5. **外部风险与交易条件**（政策风险 + 估值合理性，作为乘数）

## 优势

1. **标准化公式**：每个维度都有明确的计算公式，减少主观判断
2. **防作弊机制**：使用对数函数、分段函数等数学工具压制数据造假
3. **一票否决**：第五维度作为乘数，可以直接否决高风险项目
4. **行业对标**：强制要求与行业平均水平对比，防止盲目乐观
5. **简洁明了**：从10个维度精简到5个，更易理解和沟通
