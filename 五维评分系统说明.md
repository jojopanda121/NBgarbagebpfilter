# 五维标准化评分系统说明

## 概述

本系统采用5个核心维度，每个维度独立计算 0-100 分，最终取**简单平均**作为总分。
评分公式完全在 JS 端（`server/scoring.js`）执行，杜绝让大模型直接拍脑袋给总分。

## 总分计算

```
Total_Score = (S1 + S2 + S3 + S4 + S5) / 5
```

**五个维度等权（各 20%），不使用加权。**

---

## 五个维度

### 第一维度：时机与天花板（S1, 等权 20%）

**输入变量：**
- `TAM_Million_RMB`: 目标可触达市场规模（百万人民币，如 1 亿 = 100）
- `CAGR`: 行业预期年复合增长率（百分比数字，如 25 表示 25%）

**计算公式：**
```
tamScore = min(60, round(17.5 × log10(max(1, TAM))))
cagrScore = min(40, CAGR)
S1 = min(100, tamScore + cagrScore)
```

**设计说明：**
- 对数函数在 TAM = 3000（30亿人民币）时得到约 60 分，避免用百亿大市场旧标准误杀垂直赛道
- CAGR 线性贡献，上限 40 分
- 数据缺失兜底：TAM 默认 1，CAGR 默认 0

---

### 第二维度：产品与壁垒（S2, 等权 20%）

**输入变量：**
- `TRL`: 技术就绪水平（1-9 级，9 为可量产）
- `Competitor_Rank_Score`: 竞品排名评分（1-10，LLM 基于检索输出）

**计算公式：**
```
S2 = round(0.4 × (TRL / 9 × 100) + 0.6 × (Rank × 10))
```

**竞品排名评分说明：**
- 8-10 分: 行业 Top 5 且极难复制
- 4-7 分: 腰部或细分第一
- 1-3 分: 红海同质化跟风者

**数据缺失兜底：** TRL 默认 3，Rank 默认 5

---

### 第三维度：资本效率与规模效应（S3, 等权 20%）

**输入变量：**
- `Industry_Capital_Score`: 行业资本效率评分（1-10，LLM 输出）
- `Industry_Scale_Score`: 行业规模效应评分（1-10，LLM 输出）

**计算公式：**
```
S3 = Capital_Score × 5 + Scale_Score × 5
```

**设计说明：**
- 针对早期项目缺乏财务数据，改由 LLM 基于顶级 VC 框架评估赛道宏观属性
- 10 分 = 纯软件 SaaS 轻资产 / 双边网络效应
- 1 分 = 重资产制造 / 人力密集型无规模效应

**数据缺失兜底：** 默认 5 分中性分（S3 = 50）

---

### 第四维度：团队基因（S4, 等权 20%）

**多因子团队评分模型（v4.1）：**

**输入变量（LLM 输出 1-10 分）：**
- `Team_Experience_Score`: 经验深度（权重 30%）
- `Team_Domain_Match_Score`: 行业匹配度（权重 25%）
- `Team_Completeness_Score`: 团队完整性（权重 20%）
- `Team_Track_Record_Score`: 过往成绩（权重 15%）
- `Team_Education_Score`: 教育背景（权重 10%）

**计算公式：**
```
weighted = 0.30 × Experience + 0.25 × DomainMatch + 0.20 × Completeness
         + 0.15 × TrackRecord + 0.10 × Education
S4 = round(weighted × 10)
```

**兼容旧接口：** 如果只有 `Founder_Exp_Years`，使用对数曲线 `min(10, 2.5 × ln(years + 1))` 计算

**数据缺失兜底：** 各子因子默认 5 分

---

### 第五维度：BP 诚信度（S5, 等权 20%）

**输入：** Agent B 声明核查结果数组（claim_verdicts）

**verdict 映射规则（满分 10）：**
| verdict | 分数 | 含义 |
|---------|------|------|
| 诚实 / 保守低估 | 10 | 正面信号 |
| 存疑 | 6 | 无罪推定，LLM 覆盖不足 |
| 夸大 | 3 | 有证据的负面信号 |
| 信息不对称 | 2 | 故意隐瞒 |
| 严重夸大 | 1 | 严重负面 |
| 证伪 | 0 | 声明明显错误 |

**计算公式：**
```
S5 = round(所有声明得分均值 × 10)
```

**无数据兜底：** 60 分（刚好及格，不惩罚）

---

## 评级标准

| 分数范围 | 评级 | 标签 | 建议行动 |
|---------|------|------|---------|
| >= 85 | **A** | 强烈推荐投资 (Fast Track) | 立刻推进：24 小时内约见创始人，启动尽调和估值建模 |
| 70-84 | **B** | 谨慎推荐 (Proceed with DD) | 安排面谈，重点考察团队和单位经济模型 |
| 60-69 | **C** | 观望跟踪 (Keep In View) | 季度跟踪，关注关键里程碑 |
| < 60 | **D** | 建议放弃 (Reject / Archive) | 结构性硬伤，归档标注否决原因 |

## 技术实现

- **评分引擎**：`server/scoring.js` — 纯 JS 数学计算
- **声明核查**：`server/services/pipelineService.js` — Agent B 批量核查
- **前端展示**：`client/src/components/` — 五边形雷达图
